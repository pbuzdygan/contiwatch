Global update policy details

          
        

        

          
Global update policy sets the default behavior for containers without a per-container label.

          
Labels override the global setting: contiwatch.policy=update, notify_only, skip.

          

            

              
                Policy behavior
              
            

            

              
notify_only: pulls metadata, detects updates, and reports them. No container changes.

              
update: detects updates and recreates the container when possible. Paused containers are skipped. Stopped containers are skipped unless “Update stopped containers” is enabled.

              
skip: excludes the container from checks and updates entirely.

            

          

          

            

              
                Examples
              
            

            

              
Global update policy = update means containers update automatically unless they are labeled skip.

              
Global update policy = notify_only reports updates, while a container labeled update still updates automatically.
